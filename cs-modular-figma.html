Modular Figma Frame

Built a modular, scalable Figma frame system to standardize design components across a multi-product digital pathology design team.

Role Senior UX Designer
Timeline 2024
Platform Figma / Design System

Overview

One design language across a fragmented product suite

Roche's digital pathology suite spans multiple interconnected products — each with its own design team, working in parallel across different geographic locations. Over time, this had produced a sprawling Figma environment where components were duplicated, naming conventions differed between files, and updating a shared element meant tracking down every instance across multiple libraries manually.

The goal of this project was to create a single, modular component system that all product designers could adopt without disrupting their existing workflows — a shared source of truth that would keep the product suite visually and functionally consistent as it scaled.

Problem

Rebuilding the same components — without a shared language

Without a unified component library, designers across the four products were effectively working in isolation. A button component in one product looked slightly different from the same component in another. Data tables, modals, and form inputs each had multiple competing versions, and there was no mechanism to propagate updates across products when design decisions changed.

This fragmentation had a compounding effect. Design reviews became longer as teams debated which version of a component was correct. Handoff documentation was inconsistent, creating confusion for engineering teams. And the cognitive overhead of managing individual component sets left designers with less time for the high-value UX work the products actually needed.

Process

Audit, architect, build, adopt

The project started with a systematic component audit across all four products — cataloguing every UI element in use, identifying duplicates, and assessing which variants were genuinely necessary versus which had drifted without purpose. This produced a clear inventory that formed the basis for consolidation decisions.

From the audit, I defined a token-based design architecture — separating primitive values (colour, spacing, radius, type scale) from semantic tokens (surface, border, interactive states) so that theming and brand updates could cascade across the entire system without component-by-component changes.

The Figma library was built with auto-layout throughout and a variant structure that mapped directly to the component API engineers would implement. Once the core library was stable, I ran a series of onboarding workshops with all 12 designers across the product teams — covering how to consume the library, how to contribute new components, and the governance process for proposing changes.

Outcomes

Faster design, consistent products

Following rollout, the shared library became the primary source of truth for all four product teams. Prototyping velocity improved significantly as designers stopped rebuilding foundations and focused on solving interaction problems. The governance process established during onboarding has continued to support healthy library evolution without central bottlenecks.

60% faster prototyping
1 shared library across 4 products
Adopted by 12 designers
